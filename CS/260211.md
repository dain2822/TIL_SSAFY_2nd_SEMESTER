- 쿠키, 세션
    - 웹에서 사용자 정보를 유지하기 위해 사용되는 기술
    - 쿠키
        - 클라이언트에 저장되는 데이터로 사용자의 로그인 정보나 방문 기록 등을 브라우저에 남김
        - 사용자가 수정하거나 탈취될 가능성이 있어 보안에 취약함
        - 파일 형태로 브라우저에 남기 때문에 속도가 빠르지만 보안이 약함
        - 브라우저 종료 후에도 유지될 수 있음
        - 용량 제한이 있음
        - 일반적으로 자동 로그인이나 단기 정보 저장에 활용
        
        ⇒ **“클라이언트 저장, 빠름”**
        
    - 세션
        - 서버에 저장되는 데이터로, 사용자가 접속한 동안에만 정보를 보관함
        - 서버 관리 방식이라 보안성이 더 높음
        - 서버 처리라 속도는 다소 느릴 수 있음
        - 일정 시간 후 또는 브라우저 종료 시 삭제되는 경우가 많음
        - 서버 메모리를 사용해 상대적으로 더 많은 데이터를 저장할 수 있음
        - 일반적으로 로그인 상태 유지 등 민감한 정보에 활용
        
        ⇒ **“서버 저장, 안전함”**
        
    - 유사 질문
        
        : 로컬 스토리지와 세션 스토리지, 쿠키의 차이
