- 메시지 큐
    
    : 시스템 간 데이터를 비동기적으로 전달하기 위해 메시지를 큐 형태로 저장하고 처리하는 구조
    
    : 서비스 간 직접 통신 대신 메시지를 큐에 저장하고, 다른 서비스가 이를 가져가 처리하는 방식
    
    : 대규모 시스템에서 서비스 간 결합도를 낮추고 안정성을 높이고 위해 사용됨
    
    - 필요한 이유
        - 서비스가 직접 호출하면 문제가 생길 수 있음
        - 예를 들어, 회원가입 → 이메일 발송 / 회원 서비스 → 이메일 서비스 직접 호출일 경우
            - 이메일 서버가 느리거나 장애 발생 시 회원가입도 같이 느려지거나 실패 가능
        
        → 서비스 간 의존성 감소
        
        → 장애 전파 방지
        
    - 동작 구조
        - **Producer**
            
            : 메시지를 생성하여 큐에 전달
            
        - **Message Queue**
            
            : 메시지를 임시 저장
            
        - **Consumer**
            
            : 큐에서 메시지를 가져와 처리
            
    - 대표적인 메시지 큐 시스템
        - **Kafka**
            
            : 대용량 데이터 처리 및 스트리밍에 강점
            
        - **RabbitMQ**
            
            : 메시지 전달 보장에 강점
            
        - **Amazon SQS**
            
            : AWS 클라우드 기반 메시지 큐
            
    - 장점
        - 서비스 간 **결합도 감소**
        - 비동기 처리 가능
        - 트래픽 급증 시 **버퍼 역할**
        - 장애 격리 가능
    - 단점
        - 시스템 구조 복잡도 증가
        - 메시지 처리 지연 가능
        - 메시지 중복 처리 문제 발생 가능
    - 활용 사례
        - 이메일 발송
        - 주문 처리
        - 로그 처리
        - 알림 서비스
        - 데이터 스트리밍

        : 동기 처리와 비동기 처리의 차이?
