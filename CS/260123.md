- 객체지향_ Object-Oriented Programming
    
    : 현실 세계의 사물이나 개념을 객체로 표현하여 프로그램을 구성하는 방식
    
    : 객체는 데이터와 그 데이터를 처리하는 메서드를 하나로 묶은 단위이며, 이를 통해 코드의 재사용성과 유지보수성을 높일 수 있음
    
    - 객체 지향의 핵심
        1. 캡슐화 : 객체 내부의 데이터를 보호하고 필요한 부분만 외부에 공개하여 안정성을 높이는 개념
        2. 상속 : 상위 클래스의 속성과 기능을 하위 클래스가 물려받아 중복을 줄이고 재사용성을 높이는 원칙
        3. 다형성 : 동일한 이름의 메서드가 상황이나 객체에 따라 다르게 동작하도록 하여 프로그램을 더 유연하게 만드는 특성
        4. 추상화 : 불필요한 세부 구현을 감추고 중요한 개념과 동작만을 드러내어 복잡성을 줄이는 원칙
    
    ⇒ 객체 지향은 캡슐화를 통해 안정성을, 상속을 통해 효율성을, 다형성을 통해 유연성을, 추상화를 통해 단순성을 확보할 수 있는 특징 덕분에 대규모 소프트웨어 개발에서 구조적이고 체계적인 설계를 가능하게 함

- RESTful API
    
    : REST는 Representational State Transfer의 약자로, 웹에서 자원을 다루는 아키텍처 스타일임
    
    : 자원을 URI(고유 식별자)로 표현하고, 자원에 대한 행위는 HTTP 메서드로 정의함
    
    ex) GET은 조회, POST는 생성, PUT은 수정, DELETE는 삭제로 매핑됨
    
    : RESTful API는 클라이언트와 서버 간의 통신을 단순하고 일관성 있게 만들어줌
    
    : 확장성, 가독성, 유지보수성이 뛰어남
    
    : 무상태(Stateless) 특성을 가져, 서버가 클라이언트 상태를 보관하지 않아 효율적임
    
    : JSON이나 XML 같은 포맷으로 데이터를 주고받을 수 있지만, 보통은 JSON을 많이 사용함
    
    ex) OPEN API, SNS 로그인 연동, 금융권 데이터 조회 API 등
    
    : 모든 상황에서 RESTful이 완벽한 것은 아니며, 실시간성이 중요한 경우 WebSocket 같은 방식이 더 적합할 수 있음
    
    ⇒ RESTful API는 자원을 직관적으로 표현하고, HTTP 규칙을 활용해 효율적으로 통신하는 표준화된 방법
    
    - 유사 질문
        
        : REST와 RESTful 비교
