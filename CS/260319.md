- Idempotency(멱등성)
    
    : 동일한 요청을 여러 번 수행해도 결과가 한 번 수행한 것과 동일하게 유지되는 성질
    
    : 주로 API 설계에서 중복 요청 처리 문제를 방지하기 위해 사용하는 개념
    
    - 필요한 이유
        - 결제 요청 → 네트워크 문제로 응답 X → 다시 요청
        - 멱등성이 없는 경우 → 결제 2번 발생
        - 멱등성이 있는 경우 → 한 번만 처리됨
    - 핵심 개념
        - 같은 요청을 여러 번 보내도 결과는 동일함
        - 서버는 중복 요청을 구분하고 한 번만 처리해야 함
    - HTTP 메서드와 멱등성
        - GET
            
            : 멱등성 보장(조회)
            
        - PUT
            
            : 멱등성 보장(같은 데이터로 덮어쓰기)
            
        - DELETE
            
            : 멱등성 보장(여러 번 삭제해도 결과 동일)
            
        - POST
            
            : 멱등성 없음(요청마다 새로운 리소스 생성)
            
    - 구현 방법
        - Idempotency Key 사용
            
            : 요청마다 고유 키를 부여하여 중복 요청 판별
            
        - DB Unique Constraint 활용
            
            : 요청마다 고유 키를 부여하여 중복 요청 판별
            
        - 요청 상태 저장
            
            : 이미 처리된 요청인지 확인
            
    - 장점
        - 중복 요청 방지
        - 데이터 일관성 유지
        - 결제/주문 시스템 안정성 확보
    - 단점
        - 추가적인 저장소 필요
        - 구현 복잡도 증가
        - 키 관리 필요
    - 유사 질문

        : POST는 왜 멱등하지 않는지?
        
        : 멱등성을 구현하는 방법
        
        : 결제 시스템에서 멱등성이 중요한 이유
