- 페이지 교체 알고리즘
    
    : 가상 메모리 시스템에서 물리 메모리가 가득 찼을 때 어떤 페이지를 제거하고 새로운 페이지를 적재할 지 결정하는 알고리즘
    
    : Page Fault 발생 시 메모리에서 제거할 페이지를 선택하기 위해 사용됨
    
    : 운영체제의 메모리 관리 성능에 큰 영향을 미치는 핵심 기법
    
    - 왜 필요한가?
        - 물리 메모리(RAM)는 제한적
        - 새로운 페이지를 메모리에 올려야 할 때
        - 기존 페이지 중 하나를 제거해야 함
        
        → 어떤 페이지를 제거하느냐에 따라 성능 차이 발생
        
    - FIFO(First In First Out)
        - 가장 먼저 메모리에 들어온 페이지를 제거
        - 구현이 간단
        - Belady’s Anomaly 발생 가능
    - LRU(Least Recently Used)
        - 가장 오랫동안 사용되지 않은 페이지 제거
        - 실제 시스템에서 많이 사용되는 방식
    - LFU(Least Frequently Used)
        - 가장 적게 사용된 페이지 제거
    - Optimal Algorithm
        - 앞으로 가장 오랫동안 사용되지 않을 페이지 제거
        - 이론적으로 가장 효율적
        - 미래 예측이 필요해 실제 구현 불가
    - Belady’s Anomaly
        
        : 페이지 프레임 수를 늘렸는데 **Page Fault가 오히려 증가하는 현상**
        
    - 장점
        - 제한된 메모리를 효율적으로 사용 가능
        - Page Fault 감소
        - 시스템 성능 향상
    - 단점
        - 알고리즘 구현 비용 존재
        - LRU 같은 경우 추가적인 관리 비용 발생
        - 잘못된 알고리즘 선택 시 성능 저하
    - 유사 질문
        
        : FIFO와 LRU의 차이?
        
        : Belady’s Anomaly란?
        
        : Optimal 알고리즘이 실제로 사용되지 않는 이유?
        
        : Page Fault가 많이 발생하면 생기는 문제?
