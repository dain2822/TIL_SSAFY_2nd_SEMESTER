- 트랜잭션 격리수준(Isolation Level)
    
    : 데이터베이스에서 여러 트랜잭션이 동시에 실행될 때 서로의 작업 결과에 대해 어느 정도까지 접근을 허용할 것인지 정의하는 기준
    
    : 트랜잭션은 ACID 특성을 가지며, 그중 Isolation은 각 트랜잭션이 독립적으로 실행되도록 보장하는 속성
    
    : 격리수준을 높이면 정합성은 좋아지지만 성능이 떨어지고, 낮추면 동시성은 좋아지지만 데이터 충돌 가능성이 증가함
    
    - Read Uncommitted
        
        : 커밋되지 않은 데이터도 읽을 수 있어 Dirty Read가 발생할 수 있음
        
    - Read Committed
        
        : 커밋된 데이터만 읽을 수 있어 Dirty Read는 방지하지만 Non-repeatable Read가 발생할 수 있음
        
    - Repeatable Read
        
        : 같은 데이터를 반복 조회해도 동일한 값을 보장하지만 Phantom Read가 발생할 수 있음
        
    - Serializable
        
        : 가장 높은 격리수준으로, 모든 트랜잭션을 순차적으로 실행하는 것처럼 동작하여 위 문제들을 모두 방지함
        
    
    : 격리 수준이 높을수록 데이터 일관성은 높아지지만 성능이 저하되기 때문에, 서비스 특성에 따라 적절한 격리수준을 선택하는 것이 중요함
