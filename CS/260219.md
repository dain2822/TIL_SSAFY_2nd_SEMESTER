- CI/CD
    
    : 지속적 통합(Continuous Integration)과 지속적 배포(Continuous Deployment 또는 Delivery)를 의미
    
    : 코드 변경 사항을 자동으로 빌드, 테스트, 배포까지 연결하는 개발 자동화 파이프라인
    
    - CI/CD 파이프라인 설계의 장점
        - 코드 변경이 자동으로 검증되고 배포되면서 개발 및 배포 속도가 빨라짐
        - 반복 작업을 자동화하며 휴먼 에러를 줄이고 일관된 품질을 유지할 수 있음
        - 테스트 단계가 파이프라인에 포함되어 있어 초기 단계에서 버그를 발견할 수 있음
        - 배포 이력과 변경 사항이 체계적으로 관리되어 롤백과 이슈 추적에 용이함
    - CI/CD 파이프라인 설계의 단점
        - 초기 구축과 설정에 시간과 비용이 소요됨
        - 파이프라인 구조가 복잡해질 수록 관리 난이도가 증가함
        - 테스트 코드와 자동화 스크립트에 대한 지속적인 유지보수가 필요함
        - 보안 설정을 충분히 하지 않으면 인증 정보 노출과 같은 위험이 발생할 수 있음
        - 소규모 프로젝트에서는 과도한 자동화가 오히려 비효율적일 수 있음
    
    ⇒ 잦은 배포가 필요한 서비스에서 특히 효과적이며, 프로젝트 규모와 조직 상황에 맞춰 단계적으로 도입하는 것이 중요함
    
    - 질문 유형
        
        : CI/CD에 대한 설명
        
        : 파이프라인 설계의 장단점
