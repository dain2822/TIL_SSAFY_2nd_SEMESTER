- JPA(Java Persistence API)
    
    : 자바에서 객체와 관계형 데이터 베이스를 매핑하는 ORM 표준 사양임
    
    : SQL을 직접 작성하지 않고, CRUD 같은 반복적인 SQL 작성 없이 메서드 호출로 처리할 수 있는 장점이 있고, 유지보수성이 높음
    
    : 테이블 구조 변경 시 SQL 대신 엔티티 클래스만 수정하면 됨
    
    : 표준 API를 사용하므로 데이터 베이스 교체 시 코드 수정이 최소화되며, 객체 지향적 데이터 접근이 가능함
    
    : 컬렉션 다루듯이 데이터 조작이 가능함
    
    : 단점으로 러닝 커브가 있음
    
    : ORM의 동작 원리와 객체-관계 패러다임 불일치를 이해해야 함
    
    : 복잡한 쿼리는 여전히 JPQL 또는 네이티브 SQL을 작성해야 함
    
    : 잘못 사용하면 성능 저하가 발생할 수 있음
    
    → JPA는 객체 중심 설계와 생산성을 높여주지만, 원리와 제약을 이해하고 사용하는 것이 중요함
    
    - 유사 질문
        
        : JPA와 Hibernate의 차이
        
    - 주요 출제 기업
        
        : 25년 상반기 엠로, 카카오페이, IM뱅크, 삼성 SDS 등

- MSA(Microservices Architecture)
    
    : 하나의 거대한 모놀리식 애플리케이션을 작은 단위의 독립적인 서비스들로 분리하여 개발,  배포, 운영하는 아키텍처
    
    : 각 서비스는 독립적으로 배포 가능하며, API를 통해 통신함
    
    : 서비스 단위로 분리되어 있어 유지보수와 확장이 용이하며, 특정 서비스에만 트래픽이 집중되더라도 부분 확장이 가능해 자원 효율성이 높음
    
    : 서비스가 많아지면 서비스 간 통신 복잡도가 증가하고, 네트워크 비용이 커짐
    
    : 분산 환경에서 트랜잭션 관리와 데이터 일관성을 보장하기 어려움 + 모니터링과 로그 관리 등 운영 관리 체계도 더 복잡해짐
    
    : MSA 아키텍처 설계 시에는 서비스 경계를 명확히 정의하는 것이 핵심임
    
    : API 게이트웨이를 통한 진입점 관리, 서비스 디스커버리, 로드 밸런싱, 장애 감지 및 복구 체계가 필요함
    
    : CI/CD 파이프라인, 컨테이너 오케스트레이션을 활용해 자동화와 확장성을 확보해야 함
    
    - 문제 유형
        
        : MSA의 장,단점
        
    - 주요 출제 기업
        
        : 25년 상, 하반기 현대오토에버, 25년 상반기 하나은행, 우리은행, LG전자, 한화시스템, 에스코어, KB데이타시스템 등
