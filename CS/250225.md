- 동기화(Synchronization)
    
    : 여러 프로세스 또는 스레드가 공유 자원에 동시에 접근할 때 발생하는 문제를 방지하기 위한 기법
    
    : 멀티스레드 환경에서 데이터 일관성과 무결성을 보장하기 위해 반드시 필요한 개념
    
    : 공유 자원(변수, 파일, DB 등)에 대한 접근을 제어하여 Race Condition을 방지하는 것이 목적
    
    - 핵심 개념
        - 임계 영역(Critical Section)
            
            : 공유 자원에 접근하는 코드 영역
            
            → 한 번에 하나의 스레드만 실행되어야 함
            
        - Race Condition(경쟁 상태)
            
            : 여러 스레드가 동시에 접근하여 예상과 다른 결과가 발생하는 현상
            
        - 상호 배제(Mutual Exclusion)
            
            : 동시에 하나의 스레드만 자원에 접근하도록 보장하는 것
            
    - 대표적 동기화 도구
        - Mutex(뮤텍스)
            
            : 하나의 스레드만 자원 접근 가능(lock / unlock 구조)
            
            : 소유 개념 존재
            
        - Semaphore(세마포어)
            
            : 정수 카운터 기반
            
            : 여러 스레드 접근 허용 가능
            
            : 소유 개념 없음
            
        - Monitor
            
            : Java의 `synchronized` 키워드
            
            : 내부적으로 lock과 조건 변수를 포함
            
    - 장점
        - 데이터 무결성 보장
        - 멀티스레드 환경에서 안정성 확보
        - 예측 가능한 프로그램 동작 가능
    - 단점
        - 성능 저하(Lock으로 인해 병목 발생 가능)
        - Deadlock 발생 가능
        - 설계 복잡도 증가
        - 과도한 동기화 시 확장성 저하
