- MVC(Model-View-Controller) 패턴
    - 소프트웨어 설계 방식 중 하나로, 애플리케이션을 Model(데이터 및 비지니스 로직), View(화면 표시), Controller(입력 제어) 세 영역으로 분리하여 관리하는 구조
    - **Model**은 핵심 데이터와 처리 로직을 담당하여 데이터베이스와 직접 상호작용함
    - **View**는 사용자가 보게 되는 화면으로, Model에서 전달된 데이터를 기반으로 UI를 구성함
    - **Controller**는 사용자의 요청이나 입력을 전달받아 적절한 Model을 호출하고 처리 결과를 View로 전달함
    - 역할을 명확히 분리하기 때문에 유지보수와 기능 확장에 용이하고, 협업에도 효율적이라는 장점이 있음
    - UI 변경이 비지니스 로직에 영향을 거의 미치지 않아 변경에 유연함
    - 예를 들어, 웹 요청이 발생하면 Controller가 이를 처리한 후 Model에서 필요한 정보를 받아 View로 전달하는 흐름을 가짐
    - React와 같은 프론트엔드 프레임워크는 완전한 MVC 구조는 아니지만 View 중심 패턴으로 MVC의 영향을 받은 사례임
    - MVC는 웹 개발뿐만 아니라 다양한 애플리케이션 설계에 활용되는 대표적인 디자인 패턴임
