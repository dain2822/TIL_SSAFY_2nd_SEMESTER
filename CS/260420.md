- 로드 밸런싱
    
    : 여러 서버에 들어오는 네트워크 트래픽을 효율적으로 분산하여 시스템의 성능과 가용성을 높이는 기술
    
    : 특정 서버에 부하가 집중되는 것을 방지하고, 장애 발생 시에도 다른 서버로 요청을 우회시켜 서비스의 안정성을 확보하는 것이 목적
    
    → 이런 로드밸런싱은 로드밸런서라는 장비나 소프트웨어를 통해 수행됨
    
    : 로드밸런서는 클라이언트의 요청을 받아 여러 서버 중 하나로 전달
    
    : 어떤 서버로 요청을 보낼지를 결정하기 위해 다양한 알고리즘이 사용됨
    
    - 라운드 로빈 방식
        
        : 요청을 순차적으로 각 서버에 균등하게 분배하는 방식
        
    - 가중 라운드 로빈
        
        : 서버의 성능에 따라 가중치를 부여해 더 높은 성능의 서버에 더 많은 요청을 할당함
        
    - 최소 연결 방식
        
        : 현재 연결 수가 가장 적은 서버로 요청을 보내 동적인 부하 분산이 가능함
        
    - 가중 최소 연결 방식
        
        : 최소 연결 방식 + 서버 성능까지 반영한 알고리즘
        
    - IP 해시 방식
        
        : 클라이언트의 IP를 기준으로 특정 서버에 고정적으로 연결되도록 하여 세션 유지에 유리함
        
    
    : 로드 밸런싱 알고리즘은 단순 분산, 서버 성능 고려, 세션 유지 등 다양한 목적에 따라 선택되며, 서비스 특성에 맞는 적절한 알고리즘을 사용하는 것이 중요함
    
    - 질문 유형
        
        : 로드밸런싱, 로드밸런싱 알고리즘 나열, 로드밸런서의 L4, L7 차이 등
