- 프로세스(Process) vs. 스레드(Thread)
    - 프로세스(Process)
        - 운영체제로부터 자원을 할당받아 실행 중인 프로그램
        - 실행 파일이 메모리에 올라가 실행되는 순간 → 프로세스
        - 독립된 메모리 공간을 가짐(Code, Data, Heap, Stack)
        - 다른 프로세스와 메모리를 공유하지 않음
        - 하나 이상의 스레드를 가질 수 있음
        - 안정성이 높음(한 프로세스가 죽어도 다른 프로세스에 영향이 없음)
        - 생성 비용이 큼
        - 문맥 교환(Context Switching) 비용이 큼
    - 스레드(Thread)
        - 프로세스 내부에서 실행되는 작업의 단위
        - 같은 프로세스 내에서 Heap, Code, Data 공유함
        - Stack은 각각 독립됨
        - 생성 비용이 작고 실행이 빠름
        - 자원 공유가 쉬움
        - 동기화 문제가 발생하기도 함(하나가 죽으면 프로세스 전체에 영향을 줄 수 있음)

    |구분|프로세스|스레드|
    |---|---|---|
    |메모리|독립|공유|
    |생성비용|큼|작음|
    |안정성|높음|낮음|
    |통신|IPC 필요|그냥 공유 변수|
