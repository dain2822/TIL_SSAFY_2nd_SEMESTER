- OAuth 2.0
    
    : 사용자 인증 정보를 직접 공유하지 않고, 제 3자 애플리케이션이 사용자 대신 자원에 접근할 수 있도록 권한을 위임하는 인증 프레임워크
    
    : 소셜 로그인에 사용되는 표준 방식
    
    - 필요한 이유
        - 기존에는 서비스가 사용자 비밀번호;를 직접 받아 보안의 위험 존재
        - OAuth 방식은 사용자 비밀번호는 제공자(구글, 카카오 등)만 알고 서비스는 토큰만 받아 인증 → 보안성 향상
    - 주요 구성 요소
        - Resource Owner : 사용자
        - Client : 로그인 기능을 제공하는 서비스
        - Authorization Server : 인증 담당(구글, 카카오 등)
        - Resource Server : 사용자 정보를 제공하는 서버
    - 동작 흐름
        1. 사용자가 소셜 로그인 클릭
        2. 인증 서버(구글 등)로 이동
        3. 로그인 및 권한 승인
        4. Authorization Code 발급
        5. 서버에서 Access Token 요청
        6. Access Token으로 사용자 정보 요청
    - 장점
        - 사용자 비밀번호를 직접 저장하지 않음
        - 보안성 향상
        - 간편한 로그인 경험 제공
        - 다양한 플랫폼 연동 가능
    - 단점
        - 구현 복잡도 높음
        - 외부 인증 서버 의존
        - 토큰 관리 필요
