- 사가 패턴(Saga Pattern)
    
    : 분산 시스템에서 하나의 트랜잭션을 여러 서비스에 걸쳐 수행할 때, 각 단계별로 로컬 트랜잭션을 실행하고 문제가 발생하면 보상 작업을 통해 전체를 롤백하는 방식
    
    : 전통적인 DB의 ACID 트랜잭션을 MSA 환경에서 대체하기 위한 패턴
    
    - 필요한 이유
        
        : 모놀리식 구조에서는 하나의 DB 트랜잭션으로 처리 가능
        
        : MSA에서는 주문 서비스 → 결제 서비스 → 배송 서비스로 각각 DB가 다름
        
        → 하나로 묶는 트랜잭션이 불가하기 때문에 사용
        
    - 코레오그래피(Choreography) 방식
        
        : 중앙 제어 없이 각 서비스가 이벤트 기반으로 동작
        
        - 장점 : 단순, 느슨한 결합
        - 단점 : 흐름 파악 어려움
    - 오케스트레이션(Orchestration) 방식
        
        : 중앙 컨트롤러가 전체 흐름을 관리
        
        - 장점 : 흐름 명확
        - 단점 : 중앙 의존성 증가
    - 장점
        - 분산 환경에서 트랜잭션 처리 가능
        - 서비스 독립성 유지
        - 확장성 우수
    - 단점
        - 구현 복잡도 매우 높음
        - 데이터 일관성 관리 어려움
        - 보상 로직 설계 필요
    - 유사 질문
        
        : Choreography와 Orchestration의 차이?
        
        : 분산 트랜잭션을 어떻게 처리?
        
        : 보상 트랜잭션이란?
