- SSR(Server Side Rendering) / CSR(Client Side Rendering)
    
    : 웹 애플리케이션에서 화면을 어디에서 렌더링하느냐에 대한 방식의 차이임
    
    - SSR
        
        : 서버에서 HTML을 완성해서 브라우저에 전달하는 방식
        
        : 사용자가 페이지를 요청하면 서버가 데이터를 조회하고, 완성된 HTML을 만들어 응답
        
        : 초기 로딩 시 바로 콘텐츠가 보이고, 검색엔진 최적화(SEO)에 유리하다는 장점이 있음
        
        : 페이지 이동이 발생할 때마다 서버와 통신해야 하므로 서버 부하가 커질 수 있고, 인터랙션이 많은 서비스에서는 체감 속도가 느릴 수 있음
        
    - CSR
        
        : 최초에 기본적인 HTML과 JavaScript 파일을 내려받은 뒤, 이후의 화면 렌더링을 브라우저에서 수행하는 방식
        
        : 초기 로딩은 상대적으로 느릴 수 있지만, 한 번 로드된 이후에는 필요한 데이터만 API로 받아와 화면을 동적으로 갱신하기 때문에 사용자 경험이 부드러움
        
    
    →  SSR은 초기 로딩 속도와 SEO에 강점, CSR은 사용자 인터랙션과 부드러운 화면 전환에 강점
    
    → 최근 두 방식의 장점을 결합한 하이브리드 방식, 초기에는 SSR로 렌더링하고 이후에는 CSR로 동작하는 구조도 많이 활용되고 있음
