- Virtual Memory(가상 메모리)
    
    : 실제 물리 메모리(RAM)보다 더 큰 메모리를 사용하는 것처럼 보이게 하는 메모리 관리 기법
    
    : 프로세스마다 독립적인 가상 주소 공간을 제공하여 메모리 보호와 효율적인 자원 관리를 가능하게 함
    
    : 현대 운영체제의 핵심 기능
    
    - 왜 필요한가?
        - RAM 용량은 제한적
        - 여러 프로그램을 동시에 실행해야 함
        - 프로그램마다 독립된 메모리 공간 필요
        - 메모리 보호 필요
        
        → 이를 해결하기 위해 가상 주소에서 물리 주소 변환 구조 사용
        
    - 가상 주소(Virtual Address)
        
        : 프로세스가 보는 주소
        
    - 물리 주소(Physical Address)
        
        : 실제 RAM 주소
        
    - MMU(Memory Management Unit)
        
        : 가상 주소를 물리 주소로 변환하는 하드웨어
        
    - Paging
        
        : 메모리를 고정 크기 블록으로 나누는 방식
        
        - 가상 메모리 → Page
        - 물리 메모리 → Frame
        
        → Page Table을 통해 매핑
        
    - 장점
        - 메모리 보호 가능
        - 프로그램 간 독립성 보장
        - 실제 RAM보다 큰 프로그램 실행 가능
        - 메모리 효율적 사용
    - 장점
        - 주소 변환 오버헤드
        - Page Fault 발생 시 성능 저하
        - Page Table 메모리 사용 증가
    - Page Fault
        
        : 필요한 페이지가 물리 메모리에 없을 때 → 디스크에서 가져옴(느림)
