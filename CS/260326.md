- 인덱스(Index)
    
    : 데이터베이스에서 검색 속도를 향상시키기 위해 사용하는 자료구조
    
    : 특정 컬럼에 대한 값을 미리 정렬된 형태로 저장하여, 전체 데이터를 탐색하지 않고 빠르게 조회할 수 있도록 도와주는 구조
    
    - 필요한 이유
        
        : 특정 값 위치를 빠르게 찾을 수 있음 → 검색 속도 크게 향상
        
    - 동작 방식
        - 일반적으로 B-Tree 구조 사용
        - 데이터를 정렬된 형태로 저장
        - 이진 탐색 기반으로 빠르게 검색
    - 장점
        - 조회 속도 향상
        - 대용량 데이터 처리에 유리
        - 검색 성능 최적화
    - 단점
        - INSERT / UPDATE / DELETE 성능 저하
        - 추가 저장 공간 필요
        - 잘못 설계하면 오히려 성능 저하
    - 종류
        - Primary Index
            
            : 기본 키 기반 인덱스
            
        - Secondary Index
            
            : 일반 컬럼에 생성
            
        - Composite Index
            
            : 여러 컬럼을 조회한 인덱스
            
        - Unique Index
            
            : 중복 허용하지 않음
            
    - 유사 질문
        
        : 인덱스를 사용하면 왜 빠른가?
        
        : 인덱스의 단점?
        
        : 어떤 컬럼에 인덱스를 걸어야 하는가?
        
        : B-Tree 구조를 사용하는 이유?
