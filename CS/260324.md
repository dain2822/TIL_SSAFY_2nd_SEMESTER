- 이벤트 기반 아키텍처
    
    : 시스템에서 발생하는 이벤트를 기반으로 서비스 간 통신을 수행하는 아키텍처 패턴
    
    : 한 서비스에서 이벤트가 발생하면, 이를 다른 서비스들이 구독하여 비동기적으로 처리하는 구조
    
    : 서비스 간 느슨한 결함을 가능하게 하는 핵심 설계 방식
    
    - 필요한 이유
        
        : 기존의 동기 호출 A 서비스 → B 서비스 → C 서비스 에서 하나라도 느리거나 장애가 나면 전체 영향을 줌
        
    
    : A 서비스 → 이벤트 발생 → 메시지 큐 → B, C 서비스 처리
    
    → 서비스 간 직접 의존성 없음
    
    → 장애 전파 최소화
    
    - 특징
        - 비동기 처리
        - 서비스 간 결합도 낮음
        - 확장성이 뛰어남
        - 이벤트 기반 흐름
    - 장점
        - 확장성 높음
        - 장애 격리 가능
        - 유연한 서비스 추가 가능
        - 높은 처리량
    - 단점
        - 디버깅 어려움
        - 데이터 일관성 관리 어려움
        - 이벤트 순서 보장 문제
        - 시스템 복잡도 증가
    - 유사 질문
        
        : 동기 방식과 비동기 방식의 차이?
        
        : 메시지 큐와 이벤트 기반 아키텍처의 관계?
        
        : 데이터 일관성 문제 해결 방법?
