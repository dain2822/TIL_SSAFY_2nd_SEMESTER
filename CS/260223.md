- 온프레미스 / 클라우드
    - IT 인프라를 어디에 두고 어떻게 운영하느냐의 차이라고 볼 수 있음
    - 온프레미스
        - 기업이 서버, 네트워크, 스토리지 같은 인프라를 직접 구매해서 사내 전산실에 구축하고 운영하는 방식
        - 초기 구축 비용이 크고, 하드웨어 관리나 장애 대응을 모두 내부 인력이 책임져야 함
        - 데이터가 내부에 존재하기 때문에 보안이나 통제 측면에서 유리함
    - 클라우드
        - 외부 클라우드 서비스 제공업체의 인프라를 인터넷을 통해 사용하는 방식
        - 필요한만큼 자원을 즉시 확장하거나 축소할 수 있어 유연성이 높음
        - 초기 투자 비용이 적고, 사용한 만큼 비용을 지불하는 구조라 비용 효율성이 좋음
        - 외부 환경에 의존하기 때문에 보안 정책이나 규제 준수가 중요한 경우에는 제한이 있을 수 있음
    
    ⇒ 기업의 보안 요구사항, 비용 구조, 서비스 확장성 등을 종합적으로 고려해 결정

        : 단독으로도 묻지만, 두가지를 비교하는 질문이 주로 출제
