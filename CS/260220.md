- Array, Linked List
    - 데이터를 저장하는 방식과 성능 특성이 서로 다른 자료구조
    - Array
        - 데이터가 연속된 메모리 공간에 저장되는 구조
        - 인덱스를 통해 특정 위치의 데이터에 바로 접근할 수 있음
        - 조회 시간 복잡도 매우 빠름
        - 중간에 데이터를 삽입하거나 삭제할 경우, 뒤에 있는 모든 데이터를 한 칸씩 이동해야함
        - 대부분 생성 시 크기가 고정되어 확장성이 제한됨
    - Linked List
        - 각 데이터가 노드 형태로 저장되고, 각 노드는 다음 노드를 가리키는 포인터를 가지고 있음
        - 메모리에 연속적으로 저장되지 않기 때문에 인덱스 기반 접근이 불가능함
        - 특정 위치의 데이터를 찾으려면 순차 탐색이 필요함
        - 삽입이나 삭제 시에는 해당 노드의 앞뒤 포인터만 변경하면 되므로, 위치만 알고 있다면 O(1)로 처리할 수 있음
        - 크기를 동적으로 조절할 수 있어 유연성이 높음
    
    ⇒ 조회와 인덱스 접근이 중요한 경우에는 Array, 데이터 추가와 삭제가 잦은 경우에는 Linked List가 적합함
    
    - 질문 유형
        
        : Array와 Linked List의 자료구조의 기본 구조 차이를 묻는 형식 출제
        
        : ArrayList와 LinkedList를 비교하는 것도 출제
