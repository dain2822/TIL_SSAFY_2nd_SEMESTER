- CAP 정리
    
    : 분산 시스템에서 일관성, 가용성, 파티션 허용성 세 가지를 동시에 모두 만족할 수 없다는 이론
    
    : 세 가지 특성 중 최대 두 가지만 선택할 수 있다는 분산 시스템 설계 원칙
    
    : 대규모 분산 데이터베이스와 클라우드 시스템 설계에서 중요한 개념
    
    - CAP의 세 가지 요소
        - Consistency(일관성)
            
            : 모든 노드가 동일한 데이터를 조회해야 함
            
            → 어떤 노드에 요청해도 같은 결과를 반환
            
        - Availability(가용성)
            
            : 모든 요청에 대해 항상 응답을 반환해야 함
            
            → 일부 노드가 장애가 있어도 서비스 응답 가능
            
        - Partition Tolerance(파티션 허용성)
            
            : 네트워크 장애로 노드 간 통신이 끊어져도 시스템이 계속 동작해야 함
            
    - 중요한 이유
        
        : 분산 시스템에서는 네트워크 장애(Partition)가 반드시 발생할 수 있음
        
        → 실제 시스템에서는 C + P 선택 혹은 A + P 선택 구조로 설계함
        
    - CAP 조합 예시
        - CP 시스템
            - 일관성 보장
            - 일부 요청 실패 가능
        - 예시
            - HBase
            - MongoDB(일부 설정)
    - AP 시스템
        - 항상 응답 가능
        - 데이터 일관성은 나중에 맞춤
            
            ex) Cassandra, DynamoDB
            
    - 장점
        - 분산 시스템 설계 기준 제공
        - 시스템 특성에 맞는 아키텍처 선택 가능
    - 단점
        - 세 가지 모두 만족 불가능
        - 상황에 따른 트레이드오프 발생
