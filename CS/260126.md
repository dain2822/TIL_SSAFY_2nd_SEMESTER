- SOLID 원칙
    
    : 객체지향 설계에서 유지보수성과 확장성을 높이는 다섯 가지 핵심원칙
    
    1. 단일 책임 원칙(SRP)
        
        : 클래스가 하나의 책임만 가지도록 설계하는 원칙
        
        : 여러 기능이 섞여 있으면 수정 시 다른 기능에 영향을 줄 수 있으므로 이를 방지함
        
    2. 개방-폐쇄 원칙(OCP)
        
        : 기존 코드를 변경하지 않고 새로운 기능을 추가할 수 있어야 한다는 원칙
        
        : 기능 확장 시 안정성을 유지할 수 있음
        
    3. 리스코프 치환 원칙(LSP)
        
        : 자식 클래스가 부모 클래스를 완전히 대체할 수 있어야함
        
        : 상속 구조에서 예측 가능한 동작을 보장함
        
    4. 인터페이스 분리 원칙(ISP)
        
        : 한 인터페이스가 너무 많은 기능을 담지 않고, 사용하는 클라이언트에 맞게 잘게 나누어져야 한다는 원칙
        
        : 불필요한 의존성을 줄일 수 있음
        
    5. 의존 역전 원칙(DIP)
        
        : 상위 모듈이 하위 구현체에 불필요한 의존을 하지 않고, 추상화된 인터페이스에 의존해야 한다는 원칙
        
        : 모듈 간 결합도를 낮추고 유연성을 확보할 수 있음
        
    
    → 코드 수정 시 버그 가능성이 줄고 테스트가 쉬워지며, 협업 시 코드 구조가 명확해짐
    
    ⇒ 실제 개발 과정에서 코드 품질을 높이는 실질적인 가이드라인
    
    - 주요 출제 기업
        
        : 25년 하반기 현대오토에버, LIG 넥트원, 25년 상반기 엠로 등
        
        
- Multi-thread
    
    : 멀티스레딩은 하나의 프로세스 안에서 여러 스레드를 생성해 동시에 작업을 처리하는 기법
    
    : 스레드는 프로세스 내에서 실행되는 가장 작은 단위로, 메모리와 자원을 공유하면서 실행됨
    
    : 멀티스레딩을 사용하면 CPU의 활용도를 높이고, 동시에 여러 작업을 병렬적으로 처리할 수 있음
    
    ex) 게임 프로그램에서 화면을 그리는 스레드와 배경 음악을 재생하는 스레드가 동시에 동작할 수 있음
    
    : 응답 속도가 빨라지고 자원 활용이 효율적임
    
    : 동기화 문제와 교착상태(deadlock) 같은 위험이 존재해 주의 깊은 설계가 필요함
    
    : 멀티프로세스와 비교하면 메모리를 공유하기 때문에 문맥 전환 비용이 적지만, 안정성 측면에서는 더 취약할 수 있음
    
    : 자바, 파이썬 C++ 등 대부분의 언어에서 멀티스레딩을 지원하는 라이브러리를 제공함
    
    : 최근 멀티코어 CPU가 보편화되면서 멀티스레딩의 중요성이 더욱 커짐
    
    - 유사 질문
        
        : 멀티프로세스와 멀티스레드의 비교
        
    - 주요 출제 기업
        
        : 25년 하반기 엠코테크놀로지 코리아, 25년 상반기 신세계 I&C, SOOP, LIG 넥스원
