- 일관성 vs. 가용성
    
    : 분산 시스템에서 데이터의 일관성과 서비스 가용성 사이에서 어떤 것을 우선할 지 선택해야 하는 개념
    
    : CAP Theorem에서 파생된 실제 설계 상황에서의 핵심 트레이드 오프
    
    - 일관성(Consistency)
        
        : 모든 사용자가 항상 같은 데이터를 조회
        
    - 가용성(Availability)
        
        : 시스템이 항상 응답을 반환
        
    
    : 분산 시스템에서는 네트워크 장애가 발생하면 둘 중 하나를 포기해야 함
    
    - Consistency 우선(CP 시스템)
        
        : 데이터 정확성이 중요한 경우
        
        : 금융 시스템, 결제 시스템, 재고 관리
        
        : 일부 요청 실패 가능
        
        : 데이터는 항상 정확해야 함
        
        ex) 은행 계좌 잔액 → 틀리면 안 됨
        
    - Availability 우선(AP 시스템)
        
        : 서비스 지속성이 중요한 경우
        
        : SNS, 검색 서비스, 로그 시스템
        
        : 항상 응답
        
        : 데이터는 나중에 일관성을 맞춤
        
        ex) 인스타 좋아요 수 → 약간 늦게 반영되어도 괜찮음
