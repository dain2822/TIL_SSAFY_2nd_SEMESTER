- CPU Scheduling(CPU 스케줄링)
    
    : 여러 프로세스 또는 스레드 중에서 어떤 작업이 CPU를 할당할지 결정하는 운영체제의 정책
    
    : 한정된 CPU 자원을 효율적으로 사용하기 위해 실행 순서를 관리하는 기법
    
    : 시스템의 성능, 응답 속도, 공정성에 직접적인 영향을 줌
    
    - 스케줄링 목적
        - CPU 활용도 증가
        - 처리량(Throughput) 증가
        - 응답 시간(Response Time) 최소화
        - 대기 시간(Waiting Time) 최소화
        - 공정성(Fairness) 보장
    - 주요 스케줄링 알고리즘
        - FCFS(First Come First Serve)
            
            : 먼저 온 프로세스를 먼저 처리
            
            : 구현 간단
            
            : Convoy Effect 발생 가능
            
        - SJF(Short Job First)
            
            : 실행 시간이 가장 짧은 작업 먼저 실행
            
            : 평균 대기 시간 최소
            
            : 실행 시간 예측 어려움
            
        - Priority Scheduling
            
            : 우선순위가 높은 프로세스 먼저 실행
            
            : 낮은 우선순위 기아(Starvation) 가능
            
        - Round Robin(RR)
            
            : 일정 시간(Time Quantum)씩 번갈아 실행
            
            : 시분할 시스템에 적합
            
            : Quantum이 너무 크면 FCFS처럼 동작, 너무 작으면 Context Switching 증가
            
        - Multilevel Queue / Multilevel Feedback Queue
            
            : 여러 개의 큐를 두고 우선순위 조정
            
            : 실제 현대 OS에서 사용
            
    - 장점
        - 시스템 자원 효율적으로 사용 가능
        - 응답성 향상 가능
        - 작업 특성에 맞는 정책 적용 가능
    - 단점
        - 알고리즘에 따라 기아(Starvation) 발생 가능
        - Context Switching 비용 발생
        - 설계 복잡도 증가
    - 선점 vs. 비선점
        - 선점(Preemptive)
            
            : 실행 중인 프로세스를 중단하고 다른 프로세스에 CPU 할당 가능
            
            예) Round Robin(RR)
            
        - 비선점(Non-preemptive)
            
            : 한 번 CPU를 할당받으면 끝날 때까지 실행
            
            예) FCFS(First Come First Serve)
            
    - 유사 질문
        
        : 선점 스케줄링과 비선점 스케줄링의 차이?
        
        : Round Robin의 Time Quantum이 너무 크거나 작으면 발생하는 문제?
        
        : SJF가 평균 대기 시간을 최소로 만드는 이유?
        
        : Context Switching이란?
