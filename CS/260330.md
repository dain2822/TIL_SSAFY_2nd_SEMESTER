- CQRS(Command Query Responsibility Segregation)
    
    : 데이터를 변경하는 작업(Command)과 조회하는 작업(Query)를 분리하는 아키텍처 패턴
    
    : 읽기와 쓰기의 책임을 분리하여 성능, 확장성, 유지보수성을 향상시키는 설계 방식
    
    - 필요한 이유
        - 하나의 DB에서 조회 + 수정 + 삭제 모두 처리
            
            → 읽기 요청이 많으면 쓰기 영향
            
            → 복잡한 쿼리 증가
            
            → 확장 어려움
            
    - 동작 구조
        1. Command → Write DB에 데이터 변경
        2. 이벤트 발생
        3. Read DB에 반영(비동기)
        4. Query는 Read DB에서 조회
    - 장점
        - 읽기 성능 최적화 가능
        - 쓰기 성능 최적화 가능
        - 확장성 뛰어남
        - 복잡한 쿼리 단순화
    - 단점
        - 구현 복잡도 높음
        - 데이터 일관성 문제
        - 데이터 동기화 필요
