- 도커(Docker)
    
    : 컨테이너 기반 가상화 기술로, 소프트웨어를 실행하는 환경을 이미지 형태로 패키징해 어디서든 동일하게 실행할 수 있게 도와주는 도구
    
    : 기존의 VM과 달리 OS 전체를 가상화하지 않고, 필요한 라이브러리와 실행 환경만을 격리해 훨씬 가볍고 빠름
    
    - 장점
        - 개발자가 만든 애플리케이션을 컨테이너에 담으면, 환경 차이 없이 로컬, 테스트, 운영 서버에서 동일하게 실행할 수 있는 장점이 있음
        - 컨테이너는 서로 독립적으로 구동되어 배포, 확장, 복구가 유연함
        - 허브를 통해 다양한 이미지를 쉽게 활용할 수 있음
    - 단점
        - 컨테이너는 운영체제 커널을 공유하기 때문에 완벽한 보안 격리가 어렵고, 보안 취약점 관리가 필요함
        - 도커만으로는 복잡한 오케스트레이션이 어려워, 쿠버네티스 같은 추가적인 관리 도구가 필요한 경우가 많음
    - 유사질문
        
        : 도커와 쿠버네티스의 차이점

- NoSQL과 RDBMS
    
    : NoSQL_ 전통적인 관계형 데이터베이스와 달리 정형화되지 않은 데이터를 유연하게 저장하고 처리할 수 있는 비관계형 데이터베이스
    
    : RDBMS_ 데이터를 테이블 형태로 저장하고, 각 테이블 간의 관계를 기반으로 SQL 언어를 사용해 데이터를 조작함
    
    : 데이터 저장 방식과 처리 방식에 큰 차이가 있음
    
    : RDBMS_ 테이블 형태의 정형 데이터 저장하며, 고정된 스키마와 SQL 언어 사용
    
    : NoSQL_ 비정형 또는 반정형 데이터 저장하며, 문서, 키-값, 컬럼, 그래프 등 다양한 형태를 지원하며, 스키마가 유연하고, 수평 확장이 쉬워 대용량 처리에 적합함
    
    → 정형 데이터를 안정적으로 관리해야 하는 금융 시스템에는 RDBMS
    
    → 실시간 로그 분석이나 사용자 피드처럼 빠르고 유연한 처리가 필요한 경우에는 NoSQL
