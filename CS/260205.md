- WEB 서버와 WAS
    - 웹 서버(Web Server)
        - 정적인 콘텐츠(HTML, CSS, 이미지 등)를 클라이언트에게 제공하는 역할
        - 대표적인 웹 서버로는 Apache, Nginx 등이 있으며, 요청이 들어오면 미리 저장된 정적 파일을 그대로 응답으로 돌려줌
        - 단순한 요청-응답 중심의 구조로, 로직이 없는 정적 리소스 처리에 적합함
    - WAS(Web Application Server)
        - 동적인 콘텐츠를 생성하는 서버로, 웹 서버와 애플리케이션 사이에서 비즈니스 로직을 처리함
        - 예를 들어, 로그인, 게시물 등록, 결제 같은 요청을 받아 데이터베이스 연동 및 서버 내 로직 실행 후 결과를 반환함
    
    → 보통 웹 서버와 WAS는 역할 분리를 통해 함께 사용되며, 웹 서버가 클라이언트 요청을 먼저 받아 정적 자원을 처리하거나, 동적 요청은 WAS로 전달함
    
    → 서버 부하를 효율적으로 분산시키고, 보안성과 확장성을 높일 수 있음
    
    ⇒ 웹 서버는 ‘정적 페이지 제공자’, WAS는 ‘적 로직 실행자’
