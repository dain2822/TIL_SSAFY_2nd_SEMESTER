- JVM(Java Virtual Machine)
    - 운영체제 위에서 자바 바이트코드를 해석하고 실행하는 역할을 함
    - 자바 코드는 바로 실행되는 것이 아니라 Javac 컴파일러가 .java 파일을 .class 파일로 변환하고, 이 바이트 코드를 JVM이 실생함
    
    → 이로 인해 자바는 “한 번 작성하면 어디서나 실행된다(Write Once, Run Anywhere)”는 특징을 가짐
    
    - 클래스 로더
        - 바이트코드를 메모리에 올림
    - 실행 엔진
        - 인터프린팅 또는 JIT(Just-In-Time) 컴파일을 통해 기계어로 변환해 실행함
    - 메모리 영역
        - 스택, 힙, 메서드 영역, PC 레지스터, 네이티브 메서드 스택 등으로 나뉘며, 프로그램 실행 중 필요한 데이터를 관리함
    - 운영체제 마다 다른 환경에서도 JVM이 중간에서 추상화 역할을 하기 때문에 자바는 높은 이식성을 유지할 수 있음
    - 자바 프로그램이 운영체제에 의존하지 않고 실행되도록 해주는 핵심 기반 환경
    - 유사 질문
        
        : 스택과 힙의 차이점을 묻는 꼬리질문
        
    - 주요 출제 기업
