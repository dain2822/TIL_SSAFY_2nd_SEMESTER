- JWT(JSON Web Token)
    
    : 사용자 인증과 정보 전달을 위해 사용되는 토큰 기반 인증 방식
    
    : 주로 서버와 클라이언트 간의 stateless한 인증을 구현할 때 활용됨
    
    : Header, Payload, Signature의 3가지 구조로 구성됨
    
    - Header
        
        : 토큰 타입과 해싱 알고리즘 정보가 담김
        
    - Payload
        
        : 사용자 정보나 권한 같은 클레임(Claim)이 포함됨
        
    - Signature
        
        : 토큰의 위변조 여부를 검증하기 위한 값
        
    
    : 세션과 달리 서버에 상태를 저장하지 않기 때문에 확장성이 뛰어남
    
    : 토큰이 탈취될 경우 보안에 취약할 수 있음
    
    → 토큰 만료 시간 설정, HTTPS 사용, Refresh Token 전략 등을 활용
    
    : 마이크로서비스 환경이나 RESTful API에서 많이 사용
    
    : 간편한 인증과 확장성을 제공하지만 보안 관리가 중요한 인증 방식
    
    - 질문 유형
        
        : JWT의 장단점
        
        : JWT의 구조
