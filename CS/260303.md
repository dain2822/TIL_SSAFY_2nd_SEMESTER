- Context Switching(문맥 교환)
    
    : CPU가 현재 실행 중인 프로세스(또는 스레드)의 상태를 저장하고, 다른 프로세스의 상태를 복원하여 실행을 전환하는 과정
    
    : 멀티태스킹 환경에서 여러 작업을 동시에 실행하는 것처럼 보이게 만드는 핵심 메커니즘
    
    - 왜 필요한가?
        
        : CPU는 한 번에 하나의 작업만 실행 가능
        
        : 여러 프로세스를 번갈아 실행해야 함
        
        : 이전 작업 상태를 저장하지 않으면 이어서 실행 불가능
        
    - 저장되는 정보(문맥, Context)
        - Program Counter(PC)
        - CPU Register 값
        - Stack Pointer
        - 프로세스 상태
        - 메모리 관련 정보
        
        → PCB(Process Control Block)에 저장됨
        
    - 동작 과정
        1. 현재 프로세스 상태를 PCB에 저장
        2. 다음 실행할 프로세스의 PCB 정보 로드
        3. CPU 실행 재개
    - 장점
        - 멀티태스킹 가능
        - 공정한 CPU 분배 가능
        - 응답성 향상
    - 단점
        - Context Switching 자체는 “오버헤드”
        - 자주 발생하면 성능 저하
        - 캐시 미스 증가 가능
    - 유사 질문
        
        : Context Switching이 많이 발생하면 왜 성능이 저하되는가?
        
        : 프로세스와 스레드의 Context Switching 차이는?
        
        : PCB에는 어떤 정보가 저장되는가?
        
        : Time Quantum이 너무 작으면 어떤 문제가 발생하는가?
