- Cache Memory(캐시 메모리)
    
    : CPU와 주기억장치(RAM) 사이에 위치한 고속 메모리
    
    : 자주 사용되는 데이터를 미리 저장해두어 CPU가 빠르게 접근할 수 있도록 하는 메모리 계층 구조의 핵심 요소
    
    : CPU 성능을 좌우하는 핵심 요소 중 하나
    
    - 왜 필요한가?
        
        : CPU 속도 >> 메모리 속도
        
        : CPU가 매번 RAM에 접근하게 되면 병목현상이 발생함
        
        → 자주 쓰는 데이터는 가까운 곳(Cache)에 두자 ⇒ 성능 향상
        
    - 지역성(Locality)
        
        : 프로그램은 특정 데이터나 명령어를 반복해서 사용하는 경향이 있음
        
        - 시간 지역성(Temporal Locality)
            
            : 최근 사용한 데이터는 다시 사용될 가능성이 높음
            
        - 공간 지역성(Spatial Locality)
            
            : 사용한 데이터 근처의 데이터도 곧 사용될 가능성이 높음
            
    - 캐시 메모리 구조
        - L1 Cache
            
            : CPU 코어 내부, 가장 빠름, 용량이 작음
            
        - L2 Cache
            
            : CPU 내부 또는 외부, 중간 속도
            
        - L3 Cache
            
            : 여러 코어가 공유, 가장 느림, 용량이 큼
            
    - 캐시 동작 방식
        - Cache Hit
            
            : 필요한 데이터가 캐시에 있음 → 빠른 접근이 가능함
            
        - Cache Miss
            
            : 캐시에 데이터가 없음 → RAM(메모리)에 접금이 필요함(성능 저하)
            
        
        → Cache Hit 비율이 높을수록 성능이 좋음
        
    - 장점
        - 프로그램 실행 속도 향상
        - CPU 대기 시간 감소
        - 시스템 전체 성능 개선
    - 단점
        - 용량이 작고 비용이 비쌈
        - Cache Miss 발생 시 성능 저하
        - Context Switching이 잦으면 캐시 효율 감소
