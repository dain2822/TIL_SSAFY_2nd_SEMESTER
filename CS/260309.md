- 웹 소켓
    
    : 웹 환경에서 클라이언트와 서버가 실시간으로 양방향 통신을 할 수 있도록 해주는 프로토콜
    
    : 일반적인 웹 통신은 HTTP 기반의 요청-응답 구조로 동작
    
    → 클라이언트가 요청을 보내야 서버가 응답을 할 수 있고 요청이 끝나면 연결이 종료되는 특징이 있음
    
    → 이런 구조에서 실시간 데이터 전달이 필요한 서비스에서 매번 요청을 보내야 하기 때문에 비효율이 발생할 수 있음
    
    : 웹 소켓은 HTTP 기반으로 핸드셰이크 과정을 통해 연결을 맺은 후 WebSocket 프로토콜로 전환됨
    
    : 연결이 한 번 수립되면 클라이언트와 서버 간의 연결이 지속적으로 유지되며, 양쪽 모두 필요할 때 데이터를 자유롭게 전송하는 것이 가능해짐
    
    → 별도의 요청 없이도 서버가 클라이언트에게 데이터를 즉시 전달할 수 있기 때문에 지연이 적고 네트워크 오버헤드가 줄어드는 장점이 있음
    
    → 주로 실시간성이 중요한 서비스에서 많이 사용됨
    
    → 대표적으로 채팅 서비스, 실시간 알림 시스템, 온라인 게임, 주식 시세나 스포츠 경기와 같은 데이터 스트리밍 서비스에서 활용
    
    - 유사 질문
        
        : 웹 소켓의 연결 과정에 대해 설명
