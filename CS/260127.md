- TCP
    - 연결형 프로토콜
    - 데이터를 전송하기 전에 3-way handshake 과정을 통해 연결을 설정함
    - 데이터의 순서와 신뢰성을 보장하며, 손실된 패킷은 재전송하여 정확한 데이터를 전달함
    - 속도가 느리지만 신뢰성이 높은 통신에 적합함
    - 흐름제어와 혼잡제어 기능을 통해 네트워크 상황에 맞춰 전송량을 조절함
    - 스트림 기반으로 데이터를 다룸
    - 주로 HTTP, FTP, SMTP와 같은 신뢰성 중심의 서비스에 사용됨
- UDP
    - 비연결형 프로토콜
    - 연결 과정 없이 데이터를 바로 전송함
    - 단순히 패킷을 보내기 때문에, 손실이 발생해도 재전송하지 않음
    - 속도는 빠르지만 신뢰성이 낮은 통신에 사용됨
    - 흐름제어와 혼잡제어 기능을 통해 네트워크 상황에 맞춰 전송량을 조절하는 제어 기능이 없음
    - 메시지 기반으로 데이터를 처리함
    - 주로 DNS, VoIP, 실시간 스트리밍처럼 속도가 중요한 서비스에 사용됨

⇒ TCP는 정확성, UDP는 빠른 응답성에 초점을 둔 프로토콜
