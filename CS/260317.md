- 요청 제한
    
    : 특정 시간 동안 클라이언트가 보낼 수 있는 요청 수를 제한하는 기술
    
    : 서버 과부하를 방지하고, 악의적인 트래픽(DDOS, 봇)을 차단하기 위해 사용됨
    
    : 안정적인 서비스 운영을 위한 핵심 트래픽 제어 기법
    
    - 필요한 이유
        - 특정 사용자가 과도한 요청을 보내면 서버 과부하 발생
        - API 남용 및 공격 가능
        
        → 요청 수를 제한해서 서버 보호, 공정한 자원 사용, 서비스 안정성 확보
        
    - 주요 알고리즘
        - Fixed Window Counter
            - 정해진 시간 단위로 요청 수 카운트
            - 구현 간단하지만 경계 문제 발생 가능
        - Sliding Window Log
            - 요청 시간을 기록하여 정확하게 제한
            - 메모리 사용량 증가
        - Sliding Window Counter
            - Fixed + Sliding 절충 방식
        - Token Bucket
            - 일정 속도로 토큰 생성 후 요청 시 차감
            - 유연한 트래픽 처리 가능
        - Leaky Bucket
            - 일정한 속도로 요청 처리
            - 트래픽을 일정하게 유지
    - 장점
        - 서버 과부하 방지
        - DDoS 공격 완화
        - 공정한 자원 분배
        - API 남용 방지
    - 단점
        - 정상 사용자 요청 차단 가능
        - 알고리즘 설계 복잡
        - 분산 환경에서 구현 어려움
    - 유사 질문
        
        : Rate Limiting이 필요한 이유?
        
        : Token Bucket과 Leaky Bucket의 차이?
        
        : Fixed Window 방식의 문제점?
        
        : 분산 환경에서 Rate Limiting을 어떻게 구현할 수 있는지?
