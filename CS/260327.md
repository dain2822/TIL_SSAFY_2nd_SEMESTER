- Redis의 특징
    
    : Redis는 인메모리 기반의 Key-Value 구조를 사용하는 NoSQL 데이터베이스로, 매우 빠른 데이터 처리 속도가 필요한 환경에서 주로 사용됨
    
    : 데이터를 디스크가 아닌 메모리에 저장하기 때문에 읽기와 쓰기 성능이 매우 뛰어남
    
    : 캐시 서버, 세션 저장소, 실시간 데이터 처리, 메시지 브로커 등 다양한 용도로 활용됨
    
    : 단순한 문자열 저장뿐만 아니라 List, Set, Hash, Sorted Set과 같은 다양한 자료구조를 지원하여 복잡한 데이터 처리에도 활용할 수 있음
    
    → 이로 인해 단순 캐시 시스템을 넘어 자료구조 기반 데이터 저장소로도 사용됨
    
    : 기본적으로 인메모리 데이터베이스이지만, RDB 스냅샷과 AOF(Append Only File)와 같은 영속성 기능을 제공하여 서버가 재시작되더라도 데이터를 복구할 수 있음
    
    → 속도와 안정성을 동시에 확보할 수 있다는 장점이 있음
    
    : 싱글 스레드 기반으로 동작하여 동시성 제어 비용이 적고, 그 대신 비동기 I/O 방식으로 높은 처리량을 유지함
    
    ⇒ 많은 요청이 동시에 들어오는 환경에서도 안정적인 성능을 유지할 수 있음
