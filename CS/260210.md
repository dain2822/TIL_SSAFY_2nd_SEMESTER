- 정규화
    - 데이터베이스에서 데이터 중복을 줄이고, 이상 현상을 방지하기 위해 테이블을 구조적으로 나누는 과정
    - 예를 들어, 한 테이블에 학생과 수강 정보를 모두 넣으면, 같은 학생 정보가 여러 번 반복 저장될 수 있음
        
        → 이렇게 중복된 데이터를 줄이기 위해 테이블을 여러 개로 분리하고, 서로 관계를 설정함
        
        → 데이터 수정이나 삭제 시에 생길 수 있는 불일치나 오류를 막을 수 있음
        
    - 단계별로 진행되며, **제 1정규형**에서는 **컬럼이 원자값**을 갖도록 하고, **제 2정규형**에서는 **부분 종속을 제거**하며, **제 3정규형**에서는 **이행 종속을 제거**함
    - 즉, 단계가 높아질수록 데이터 구조는 더 깔끔해지고 중복은 줄어듦
    - 정규화를 지나치게 하면 테이블이 너무 많아져서 조회 성능이 떨어지는 단점도 있음
    - 실제 개발에서는 성능을 고려해 비정규화를 함께 적용하기도 함
    
    ⇒ 정규화는 데이터의 무결성과 일관성을 유지하기 위한 기본적인 데이터베이스 설계 원칙임
