- 데이터 복제(Replication)
    
    : 하나의 데이터베이스 데이터를 여러 서버에 복제하여 저장하는 기술
    
    : 데이터의 가용성과 안정성을 높이고, 읽기 성능을 향상시키기 위해 사용됨
    
    - 필요한 이유
        - 단일 DB 서버만 있을 경우
            
            : 서버 장애 시 데이터 접근 불가
            
            : 트래픽 증가 시 성능 저하
            
        - Replication 사용 시
            
            : 여러 서버에 데이터 복제
            
            : 장애 발생 시 다른 서버로 대체 가능
            
            : 읽기 요청 분산 가능
            
    - 기본 구조
        - Master(Primary)
            
            : 쓰기 담당
            
        - Slave(Replica)
            
            : 읽기 담당(Master 데이터 복제)
            
    - 동작 방식
        1. Master에서 데이터 변경 발생
        2. 변경 내용을 Replica로 전달
        3. Replica에서 데이터 동기화
    - 동기 방식
        - Synchronous Replication
            
            : Master → Replica 동기화 완료 후 응답
            
            → 데이터 일관성 높음 / 속도 느림
            
        - Asynchronous Replication
            
            : Master 먼저 응답, 이후 Replica 반영
            
            → 속도 빠름 / 데이터 유실 가능
            
    - 장점
        - 가용성 향상(장애 대응)
        - 읽기 성능 향상
        - 데이터 백업 효과
        - 트래픽 분산 가능
    - 단점
        - 데이터 일관성 문제
        - 복제 지연 발생
        - 시스템 복잡도 증가
        - Master 장애 시 Failover 필요
    - 유사 질문
        
        : Master-Slave 구조 설명
        
        : 동기/비동기 복제 차이?
        
        : Replication과 Sharding의 차이?
