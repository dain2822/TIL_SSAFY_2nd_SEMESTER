- http와 https
    
    : http는 우리가 흔히 아는 웹 통신 프로토콜인데, 데이터가 암호화되지 않고 평문으로 전송됨
    
    → 누군가 네트워크를 가로채면 요청이나 응답 내용을 그대로 볼 수 있다는 큰 단점이 있음
    
    : https는 http에 TLS 보안 계층을 더한 것으로, 데이터를 주고 받을 때 암호화를 적용함
    
    → 중간에 누가 패킷을 가로채더라도 내용을 해석하기 어려움
    
    : 인증서를 통해 서버의 신원을 확인할 수 있기 때문에 피싱 사이트 같은 위장된 서버와의 연결을 방지할 수 있음
    
    : 무결성 측면에서도 데이터가 전송 중에 변조되면 감지할 수 있어 보안성이 훨씬 높음
    
    : https를 쓰려면 인증서 발급과 관리가 필요하고, 예전에는 암호화 과정 때문에 성능 부담이 있었지만, 최근에는 TLS 1.3 같은 기술 덕분에 오버헤드가 거의 없음
    
    ⇒ http는 빠르고 단순하지만 보안이 없고, https는 암호화, 인증, 무결성을 제공하는 현대 웹의 표준임
    
    - 유사 질문
        
        : REST API와 http에 대해 설명
        
        : http 2.0에 대한 질문
        
    - 주요 출제 기업
        
        : 25년 상반기 naver, 삼성전자, 한화시스템, CJ CGV, 코스콤, 옴니어스 등

- React
    
    : 자바스크립트 기반의 UI 라이브러리로, 사용자 인터페이스를 효율적으로 구축하기 위해 설계됨
    
    : 컴포넌트 기반 구조로, 화면을 작은 단위로 쪼개어 재사용성과 유지보수성을 높일 수 있음
    
    → 이렇게 쪼개진 컴포넌트는 독립적으로 관리가 가능해 확장성 있는 애플리케이션 개발을 지원함
    
    : React의 데이터 흐름은 단방향으로 관리되며, 이는 상태 변화의 추적을 용이하게 하고 디버깅을 단순화함
    
    : JSX 문법을 사용해 자바스크립트 코드 안에서 HTML과 유사한 구조를 작성할 수 있어 UI를 보다 직관적으로 표현할 수 있음
    
    : 방대한 커뮤니티와 생태계를 갖추고 있어 관련 라이브러리, 문서, 툴의 지원을 쉽게 받을 수 있음
    
    ⇒ 대규모 프론트엔드 애플리케이션 개발에 널리 활용되는 대표적인 라이브러리
    
    - 유사 질문
        
        : React와 Nextjs의 차이점
        
        : React.memo
